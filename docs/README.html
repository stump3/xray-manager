Установка

1
Клонировать репозиторий git clone или wget — один файл без внешних зависимостей
2
Запустить от root Менеджер зарегистрирует себя как системную команду
3
Установить ядро Xray Меню 1) — автоматически скачивает последнюю версию, включает BBR и Stats API
4
Добавить протокол Меню 2) — VLESS+REALITY за 30 секунд 
# Клонировать и запустить install.sh
git clone https://github.com/stump3/xray-manager.git
cd xray-manager
sudo bash scripts/install.sh

# Скрипт настроит Nginx, сертификат, Xray
# После установки — глобальная команда
sudo xray-manager
# Или скачать только скрипт
wget -qO xray-manager.sh https://raw.githubusercontent.com/stump3/xray-manager/main/xray-manager.sh
chmod +x xray-manager.sh
sudo bash xray-manager.sh

протоколы

Xray — протоколы

⭐ VLESS + TCP + REALITY
Рекомендуется Без домена XTLS Vision
Без домена и сертификата. Генерирует x25519 ключевую пару + shortId. Маскируется под реальный TLS 1.3. XTLS Vision устраняет двойное шифрование — многократный прирост скорости.
vless://UUID@IP:443?security=reality&sni=www.microsoft.com
  &fp=firefox&pbk=PublicKey&sid=shortId&spx=%2Fa3f92e1b
  &flow=xtls-rprx-vision&type=tcp&encryption=none
⚡ VLESS + XHTTP + REALITY
Без домена Новый транспорт CDN + прямое
Первый нативный транспорт Xray. Режимы: auto, packet-up, stream-up, stream-one. Header padding, XMUX. Поддерживает H2, H3 и CDN одновременно. sniffing.routeOnly:true — sniff только для маршрутизации, не переопределяет камуфляжный target.
vless://UUID@IP:443?security=reality&type=xhttp&mode=auto
  &sni=...&pbk=...&sid=...&encryption=none
☁️ VLESS + WebSocket + TLS
Нужен домен Для CDN
За Cloudflare и другими CDN. Xray слушает на localhost, Nginx проксирует WebSocket.
# Nginx
location /vless {
    proxy_pass http://127.0.0.1:PORT;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
🔄 VLESS + gRPC + TLS
Нужен домен HTTP/2
HTTP/2 мультиплексирование. В Nginx использовать grpc_pass, а не proxy_pass.
location /xray { grpc_pass grpc://127.0.0.1:PORT; }
🔄 VLESS + gRPC + REALITY
Без домена REALITY gRPC
gRPC-транспорт поверх REALITY. Не требует домена и TLS-сертификата — x25519 + shortId. flow отключён (Vision несовместим с gRPC). Пункт 12 в меню протоколов.
vless://UUID@IP:PORT?security=reality&sni=www.yahoo.com
  &fp=firefox&pbk=PublicKey&sid=shortId
  &type=grpc&serviceName=grpc&encryption=none&spx=/hex8
🌊 VLESS + SplitHTTP + TLS/H3
Нужен домен HTTP/3 QUIC CDN
Транспорт splithttp поверх QUIC. Два режима ALPN: h3 — прямое подключение, h2,http/1.1 — через CDN. Требует открытый UDP-порт.
# h3 — прямое
"alpn": ["h3"]      — Xray ↔ клиент напрямую по QUIC

# CDN-режим
"alpn": ["h2","http/1.1"]   — CDN не поддерживает H3 transport
📦 VMess + WebSocket + TLS
Нужен домен Классика
AlterId = 0 (AEAD). Широкая поддержка клиентами. Ссылка в формате vmess://base64(JSON).
🔐 Trojan + TCP + TLS
Нужен домен Маскировка HTTPS
Каждый пользователь — уникальный пароль. Трафик неотличим от обычного HTTPS.
🌑 Shadowsocks 2022
Без домена Multi-user
Метод 2022-blake3-aes-256-gcm. Два уровня паролей: серверный (мастер) + индивидуальный на каждого пользователя.

✗ Нет Masquerade

Xray — пользователи и лимиты

Имя пользователя — строковый идентификатор Xray для статистики. Допустимы короткие имена: alice, bob, user1. Реальный email не нужен, разрешены символы a-z A-Z 0-9 . _ @ -, минимум 2 символа.

ЛимитФорматТриггерХранение
По датеYYYY-MM-DDТекущее время > expire_ts.limits.json
По трафикуЧисло ГБuplink + downlink ≥ лимит.limits.json
⚡
Автопроверка: установите systemd-таймер через меню 5) Система → 4) Таймер. Проверяет лимиты каждые 5 минут. Ручной запуск: sudo xray-manager --check-limits

Статистика трафика

Данные берутся из Stats API (127.0.0.1:10085) через команду xray api statsquery. В интерфейсе отображается: ↑ uplink / ↓ downlink / = total и % от лимита.

Добавление без разрыва соединений

Пользователи добавляются и удаляются через gRPC API Xray (xray api adu / xray api rmu) — без systemctl restart. Существующие соединения не прерываются. Fallback на restart — только если Xray не активен.

spiderX — уникальный краулер per-user

Ссылки REALITY-протоколов содержат параметр spx — начальный путь краулера при обнаружении. Генерируется детерминированно из sha256(email): каждый пользователь получает уникальный путь, который не меняется между вызовами.

spx = "/" + sha256(email)[0:8]  ← /a3f92e1b для alice, /7c4d88f2 для bob
подписка

Сервер подписки

Python3-сервер на 127.0.0.1:PORT, раздаётся через Nginx или SSH-туннель. Токен в URL обязателен — сервер слушает только на localhost.

Форматы URL

GET/{token}/subBase64 — все пользователи (v2rayN, v2rayNG)
GET/{token}/clashClash YAML — все пользователи (Mihomo)
GET/{token}/u/{email}Base64 — конкретный пользователь
GET/{token}/clash/u/{email}Clash YAML — конкретный пользователь

Особенности

Profile-Update-IntervalЗаголовок настраивается от 1 до 168 ч. Смена вступает в силу при следующем запросе — перезапуск не нужен.
АвтообновлениеПри включённом флаге файлы подписки пересоздаются автоматически при каждом добавлении пользователя.
Через Nginxlocation /{token}/ { proxy_pass http://127.0.0.1:PORT; }
Через SSH-туннельssh -L PORT:127.0.0.1:PORT user@server

Миграция на новый сервер

Меню 6) MTProto → 4) Миграция. Требуется sshpass. Скрипт подключается по SSH, устанавливает telemt под нужную архитектуру/libc, копирует конфиг со всеми пользователями и секретами, запускает сервис, открывает UFW.

Port Hopping

Клиент случайно выбирает порт из диапазона при каждом подключении. Эффективно против блокировки по конкретному порту.

# config.yaml
listen: 0.0.0.0:20000-29999

# UFW открывается автоматически
ufw allow 20000:29999/udp

URI подключения

hy2://user:pass@domain:port?sni=domain&alpn=h3&insecure=0#Name

Сохраняется в /root/hysteria-<domain>.txt. Clash/Mihomo YAML — /root/hysteria-<domain>.yaml.

⚠️
Проверка DNS перед установкой: скрипт резолвит домен через dig/getent и сравнивает A-запись с IP сервера (icanhazip.com). Самая частая причина ошибок при установке — несоответствие DNS. ACME HTTP-01 challenge провалится если A-запись ещё не распространилась.

Система

BBR+

# /etc/sysctl.d/99-xray-tune.conf
net.ipv4.tcp_congestion_control  = bbr
net.core.default_qdisc           = fq
net.ipv4.tcp_fastopen            = 3       # SYN + SYN-ACK
net.ipv4.tcp_slow_start_after_idle = 0   # нет замедления после паузы
net.core.rmem_max                = 26214400 # 26 MB буфер приёма
net.core.wmem_max                = 26214400 # 26 MB буфер передачи

Бэкап и восстановление

Бэкапы сохраняются в /root/xray-backups/xray-backup-YYYYMMDD_HHMMSS.tar.gz и содержат config.json, .keys.* и .limits.json. Автоматически хранятся последние 7 архивов — старые удаляются при создании нового.

🔒
Защита при восстановлении: перед распаковкой все пути проверяются через tar -tz. Whitelist покрывает оба варианта путей в архиве: с ./-префиксом и без. Файлы за пределами /usr/local/etc/xray/ — ошибка и отмена (защита от path traversal).
